## Development vs Production

### Development (`npm run dev`)
- Demo panel visible
- Hot module replacement enabled
- Source maps included
- Console logging active
- Unminified code

### Production (`npm run build`)
- Demo panel hidden (tree-shaken)
- Optimized bundle
- Minified code
- No source maps
- Minimal console logging

## Adding New Features

### New Component
1. Create in `src/components/`
2. Import types from `src/types/`
3. Use store hooks from `src/store/`

### New Screen
1. Create in `src/screens/`
2. Import from `src/components/`
3. Add route in `App.tsx` (if routing added)

### New Order State
1. Update `OrderStatus` in `src/types/order.ts`
2. Update `getStatusColor()` in `src/components/OrderCard.tsx`
3. Update transition logic in `src/store/orderStore.ts`

### New Event Type
1. Add to `OrderEventType` in `src/types/events.ts`
2. Create event interface extending `BaseOrderEvent`
3. Add to `OrderEvent` union type
4. Handle in store and dispatcher
