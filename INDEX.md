## Documentation Principles

All documentation follows these principles:

1. **Progressive Disclosure:** Start simple, go deeper
2. **Multiple Learning Styles:** Text, diagrams, examples
3. **Task-Oriented:** Organized by "what you want to do"
4. **Scannable:** Headers, bullets, code blocks
5. **Up-to-Date:** Version and date on every file
